Analyzing Which Variables Influence Game Outcomes in a League of Legends Match

John Bang, SJ Yu

Introduction¶

Riot Games created and released League of Legends (LoL), sometimes known as League, a multiplayer online battle arena computer game in 2009. The creators of Riot were motivated to create a standalone game in the similar genre by the Warcraft III custom map Defense of the Ancients. League has been a free-to-play game since its inception in October 2009, and it earns money through the sale of customisable character skins. Both macOS and Microsoft Windows support the game.

In the game, there are two teams of five players each, and they engage in player-versus-player combat while defending and occupying their respective halves of the battlefield. The ten players each control a "champion," a figure with special abilities and a different playing style. Champions gain strength during a match by gaining experience points, earning gold, and investing in items to outwit the opposing team. In Summoner's Rift, the main game of League, a team wins by breaking through to the enemy base and destroying their "Nexus," a sizable structure within.

Riot Games provides a REST API that gives developers access to information for use in creating our own software or websites. All League of Legends APIs are now version 4 (updated on March 19, 2020). Since this release, a minor version is no longer included in the API path. Riot games API can do the following Service status, Summoner details: name, level, profile icon, account ids, Match History and Match details and timelines by Summoner name, Someone’s current game info, if someone is in the game, Other information, check out the API documentation. For this project we will be using this api to collect the data needed.

Our objective in this tutorial is to organize the in-game data that has been provided, offer insightful information about which teams are more successful in terms of champion kills, herald kills, and tower kills, and analyze which variables are correlated with game outcome based on the available data.

Data Collection¶

Throughout our program, we utilize a various amount of libraries, some of the important ones being: requests, pandas, seaborn, and statsmodel.

Unfortunately, getting the match data from a player's id requires multiple requests from the API. First, we must get the summonerId of each player. The Riot API provides us with the player information from the highest rankings - Challenger, Grandmaster and Master.

To retrieve data from the API, an API Key is required from the Riot website. After we have authorization for accessing the API, we can retrieve the summonerId of each player utilizing the requests library.

Win Rate Based on Kill Ratio¶

Here, we determine the kill ratios (team total kills/total kills among both teams) for both sides, add those kill ratios to a new column, and then count the number of games for each kill ratio. We will also count the number of wins for each kill ratio. With the total games and total wins, we can calculate the win ratio (total wins/total games) for each kill ratio.

In [ ]:
df_kills = df[['team1_champ_kills', 'team1_win', 'team2_champ_kills', 'team2_win']].copy()

# Calculate the kill ratio of team1 (team1 kills/total kills in game)
def team1_kr(row):
    total = row['team1_champ_kills'] + row['team2_champ_kills']
    return row['team1_champ_kills']/total

# Calculate the kill ratio of team2 (team2 kills/total kills in game)
def team2_kr(row):
    total = row['team1_champ_kills'] + row['team2_champ_kills']
    return row['team2_champ_kills']/total

# Create new columns with the kills ratios for each team
df_kills['team1_kill_ratio'] = df_kills.apply(team1_kr, axis=1)
df_kills['team2_kill_ratio'] = df_kills.apply(team2_kr, axis=1)

df_kills1 = df_kills[['team1_kill_ratio', 'team1_win']].copy()
df_kills2 = df_kills[['team2_kill_ratio', 'team2_win']].copy()

# Get the total number of matches for each kill ratio
team1_kd_count = df_kills['team1_kill_ratio'].value_counts()
team2_kd_count = df_kills['team2_kill_ratio'].value_counts()
kd_count = team1_kd_count.add(team2_kd_count, fill_value=0)

# Create new dataframe for kill ratios
df_kills = pd.DataFrame({
    'kill_ratio': kd_count.index, 
    'total_games': kd_count,
    }).sort_values(by=['kill_ratio'])

df_kills1 = df_kills1.groupby('team1_kill_ratio')['team1_win'].sum()
df_kills2 = df_kills2.groupby('team2_kill_ratio')['team2_win'].sum()

df_kills['total_wins'] = df_kills1.add(df_kills2, fill_value=0)
df_kills['win_rate'] = df_kills.apply(lambda row: row['total_wins'] / row['total_games'], axis=1)

Win Rate Based on Gold Ratio¶

The biggest factor in determining team strength is gold. When a team has gold, they are able to buy items that make them stronger. Here, we will calculate the gold ratio (team gold/total gold between both teams). We will have to bin the gold values into 50 ranges. For each bin, we will calculate the total number of games that have gold ratios in the bins range, the total number of wins, and finally the win rate based on the previous two values.

In [ ]:
df_gold = df[['team1_total_gold', 'team1_win', 'team2_total_gold', 'team2_win']].copy()

# Calculate the gold ratio of team1 (team1 gold/total gold in game)
def team1_gold(row):
    total = row['team1_total_gold'] + row['team2_total_gold']
    return row['team1_total_gold']/total

# Calculate the kill ratio of team2 (team2 gold/total gold in game)
def team2_gold(row):
    total = row['team1_total_gold'] + row['team2_total_gold']
    return row['team2_total_gold']/total

# Create new columns with the gold ratios for each team
df_gold['team1_gold_ratio'] = df_gold.apply(team1_gold, axis=1)
df_gold['team2_gold_ratio'] = df_gold.apply(team2_gold, axis=1)
